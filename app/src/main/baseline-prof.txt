# Baseline Profile — hand-authored cold-start hot path.
#
# Lists classes ART should AOT-compile + preload on first launch. Every line
# is a JVM-style descriptor: 'L<package/Class>;' for whole classes, or
# '<flags>L<class>;-><method>(<args>)<ret>' for individual methods.
# Flag letters:  H = hot, S = startup, P = post-startup. AGP merges this
# file into the APK at build time (assets/dexopt/baseline.prof); ART
# consumes it on install to skip JIT warm-up on the listed entries.
#
# Scope: the cold-start path from process launch → first interactive paint.
# Two top-level cases:
#   1. No server configured yet → OnboardingScreen renders.
#   2. Server configured        → CardStackScreen renders.
# Both cases share App + AppGraph + the theme + nav scaffold + the HA
# repo/settings wiring.
